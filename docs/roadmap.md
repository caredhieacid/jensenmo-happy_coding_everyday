# Roadmap

The roadmap is evidence-gated. Dates and repository size are not success metrics.

## Alpha — public foundation

- [x] one automatic coding dispatcher with three internal lanes;
- [x] progressive disclosure between `SKILL.md` and conditional references;
- [x] initial three-scenario baseline and forward routing evaluation;
- [x] installable Codex plugin manifest and remote marketplace catalog;
- [x] deterministic repository checks and release packaging;
- [x] multilingual public README and community entry points;
- [ ] run every machine-readable regression scenario in a fresh agent context;
- [ ] add disposable fixture repositories for mutation-based evaluations.

## Beta — measured behavior

- [ ] publish versioned evaluation summaries with skill commit provenance;
- [ ] measure over-orchestration, user interruption, scope drift, and verification quality;
- [ ] calibrate lane thresholds from real tasks across frontend, backend, operations, and reviews;
- [ ] verify update and uninstall flows across Codex app, CLI, and IDE skill surfaces;
- [ ] obtain native-speaker review for Japanese, Korean, and Spanish translations.

## Stable — compatibility and trust

- [ ] define compatibility promises and deprecation policy;
- [ ] publish signed or attested release artifacts;
- [ ] require the stable CI aggregate on protected branches;
- [ ] document a reproducible plugin submission and release process;
- [ ] promote from Alpha only after behavior regressions are reproducibly green.

## Explicit non-goals

- adding another top-level workflow skill;
- maximizing agent count, prompt size, or artifact count;
- claiming cross-agent compatibility before independent evaluation;
- creating a documentation website before navigation or search pressure requires one.
