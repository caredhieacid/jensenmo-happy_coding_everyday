2. **Single dispatcher** — create one implicit Codex skill with compact contracts and three evidence-based lanes.
3. **Progressive detail** — keep the hot-path rules in `SKILL.md`; move escalation and evidence detail into references.
4. **Forward test** — repeat the same scenarios in fresh contexts with the skill and close observed loopholes.
5. **Validate** — run repository structure tests, the official Codex skill validator, Markdown/link checks, and Git diff checks.
6. **Publish** — bootstrap the empty personal repository on `main`; use feature branches and pull requests afterward.
7. **Install** — link the repository skill as the unique automatic coding entrance and reversibly retire competing dispatchers.
