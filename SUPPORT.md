# Support

## Usage questions

Open a GitHub issue with the **Usage or behavior problem** form. Include the Codex surface, skill or
plugin installation method, repository instructions that may affect the workflow, and the smallest
non-sensitive prompt that reproduces the problem.

## Behavior proposals

Use the **Behavior pressure scenario** form when the workflow over-orchestrates, under-orchestrates,
mutates read-only scope, loses user constraints, or reports evidence incorrectly. A concrete scenario
is more actionable than a broad wording proposal.

## Security reports

Follow [SECURITY.md](SECURITY.md). Do not place vulnerabilities or credentials in public issues.

## Boundaries

This is a community project. It does not provide an SLA, production operations, account support, or
support for repositories and services that contributors cannot access. Help will distinguish verified
current behavior from hypotheses and older reports.
